# middle-office-excesos-regulatorios
Blog técnico sobre identificación de excesos regulatorios, falsos positivos y gestión del conocimiento en Middle Office de Inversiones.

Descripción del proyecto
Este repositorio contiene un blog técnico orientado a documentar aprendizajes, criterios y procesos relacionados con el análisis de excesos regulatorios en el Middle Office de Inversiones. El objetivo es transformar conocimiento tácito en conocimiento estructurado, accesible y versionado, fortaleciendo la autonomía profesional y la toma de decisiones basada en evidencia.

Objetivo del blog técnico
- Documentar criterios normativos y operativos utilizados en el análisis de alertas.
- Registrar aprendizajes derivados de un post-mortem constructivo.
- Reducir falsos positivos mediante documentación clara y estandarizada.
- Fortalecer la gestión del conocimiento dentro del área.
- Crear un repositorio técnico accesible mediante GitHub Pages.

Contenido del blog
El blog incluye:
- Contexto del rol del Middle Office
- Problemas recurrentes en la identificación de excesos regulatorios
- Análisis post-mortem constructivo
- Acciones implementadas
- Aprendizajes clave
- Evidencia de control de versiones
- Reflexión sobre confianza profesional y feedback
- Conclusiones finales
Todo el contenido está disponible en el archivo index.md, publicado mediante GitHub Pages.

Tecnologías utilizadas
- Markdown para documentación técnica
- GitHub para control de versiones
- GitHub Pages para despliegue del sitio
- Herramientas del Middle Office: Excel, Power Query, Bloomberg, Python, Access, Datatec, Magic, GPI, Pershing

Control de versiones
Este repositorio utiliza GitHub para registrar cambios y mejoras en la documentación.
Ejemplos de commits utilizados:
- feat: agregar entrada principal del blog técnico
- docs: incluir análisis post-mortem constructivo
- docs: añadir sección de aprendizajes clave
- style: mejorar estructura y formato del blog
- docs: añadir conclusiones finales
